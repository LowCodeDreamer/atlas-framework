# Refresh Status

Regenerates `SYSTEM-STATUS.md` with accurate counts of all system components.

## Usage

```
/refresh-status
/refresh-status --quiet
```

## Arguments

- `--quiet` — Skip verbose output, just update the file

## What It Does

1. **Counts all components:**
   - Agents in `.claude/agents/`
   - Commands in `.claude/commands/`
   - Skills in `.claude/skills/`
   - Hooks in `hooks/` (and checks which are configured)
   - Knowledge entries in `knowledge/`
   - Expertise files in `expertise/`
   - Session logs in `logs/sessions/`
   - Inbox items in `working/inbox/`
   - Active work in `working/active/`

2. **Checks integration status:**
   - MCP servers availability
   - Hook configuration in `.claude/settings.json`

3. **Identifies issues:**
   - Unconfigured hooks
   - Empty projects
   - Stale expertise

4. **Regenerates `SYSTEM-STATUS.md`**

## Process

### Step 1: Gather Counts

```bash
# Count components
AGENTS=$(ls .claude/agents/*.md 2>/dev/null | wc -l | tr -d ' ')
COMMANDS=$(ls .claude/commands/*.md 2>/dev/null | wc -l | tr -d ' ')
SKILLS=$(ls -d .claude/skills/*/ 2>/dev/null | wc -l | tr -d ' ')
HOOKS_DISK=$(ls hooks/*.sh 2>/dev/null | wc -l | tr -d ' ')
KNOWLEDGE=$(ls knowledge/*.md 2>/dev/null | grep -v "_" | wc -l | tr -d ' ')
EXPERTISE=$(find expertise -name "*.md" -not -name "_*" 2>/dev/null | wc -l | tr -d ' ')
SESSIONS=$(ls logs/sessions/*.md 2>/dev/null | wc -l | tr -d ' ')
INBOX=$(ls working/inbox/ 2>/dev/null | wc -l | tr -d ' ')
WORK_ITEMS=$(ls working/items/*.md 2>/dev/null | wc -l | tr -d ' ')
WORK_ACTIVE=$(grep -l 'status: active' working/items/*.md 2>/dev/null | wc -l | tr -d ' ')
WORK_WAITING=$(grep -l 'status: waiting' working/items/*.md 2>/dev/null | wc -l | tr -d ' ')
PLANS=$(ls working/plans/*.md 2>/dev/null | wc -l | tr -d ' ')
```

### Step 2: Check Hook Configuration

Read `.claude/settings.json` and compare to hooks on disk:

```bash
# Get configured hooks
CONFIGURED=$(cat .claude/settings.json | grep -o 'hooks/[^"]*' | sort -u)
# Compare to disk
DISK_HOOKS=$(ls hooks/*.sh | xargs -I{} basename {})
```

Identify any hooks not in settings.json.

### Step 3: Categorize Agents

Parse agent files to categorize:
- **Meta-agents:** Files matching `meta-*.md`
- **Domain experts:** Files matching `domain-*.md`
- **Judge analysts:** Files matching `judge-*.md`
- **Functional:** Everything else

### Step 4: Check Domains

```bash
# Count files per workspace
ENO_FILES=$(find projects/eno -name "*.md" -not -name "_*" 2>/dev/null | wc -l)
DOMAIN_FILES=$(find projects/<domain> -name "*.md" -not -name "_*" 2>/dev/null | wc -l)
```

### Step 5: Generate SYSTEM-STATUS.md

Write the file with all gathered data using this template:

```markdown
---
id: system_status
type: status
updated_at: {ISO_TIMESTAMP}
version: {INCREMENT}
---

# Atlas System Status

**Operational snapshot of current system state.**

*Last updated: {DATE}*

---

## Quick Health

| Metric | Count | Notes |
|--------|-------|-------|
| Knowledge entries | {KNOWLEDGE} | {note} |
| Inbox pending | {INBOX} | {status} |
| Expertise files | {EXPERTISE} | {note} |
| Session logs | {SESSIONS} | {note} |
| Work items | {WORK_ITEMS} | {WORK_ACTIVE} active, {WORK_WAITING} waiting |
| Plan files | {PLANS} | in working/plans/ |

---

## Component Inventory

### Agents ({AGENTS})

| Category | Count | Names |
|----------|-------|-------|
| Meta-agents | {count} | {names} |
| Domain experts | {count} | {names} |
| Judge analysts | {count} | {names} |
| Functional | {count} | {names} |

### Commands ({COMMANDS})

{list of commands with purposes}

### Skills ({SKILLS})

{list of skill names}

### Hooks

| Hook | Event | Configured |
|------|-------|------------|
{hook table with configured status}

---

## Known Issues

{list any issues found}

---

*Auto-generated by /refresh-status on {TIMESTAMP}*
```

### Step 6: Output Summary

Unless `--quiet`, output:

```markdown
## Status Refreshed

- **Components:** {AGENTS} agents, {COMMANDS} commands, {SKILLS} skills
- **Knowledge:** {KNOWLEDGE} entries, {EXPERTISE} expertise files
- **Health:** {issues_count} issues found
- **Updated:** SYSTEM-STATUS.md
```

## Integration

Called automatically by:
- `/retro` — End of session
- `/retro --session-end` — Full session closure

Can also be run manually anytime to verify system state.

## Example

```
User: /refresh-status

G: Refreshing system status...

## Status Refreshed

- **Components:** 17 agents, 16 commands, 13 skills
- **Knowledge:** 6 entries, 7 expertise files
- **Health:** 1 issue found
  - `log-to-n8n.sh` not configured in settings.json
- **Updated:** SYSTEM-STATUS.md

System status is current.
```
